## Codebase Patterns
- For SELF oracle fixtures, stage runtime SELF inputs from compatibility baseline `FX-SELF-001` and use only `loss.dat` as the staged spectrum source; staging oracle rewritten `loss.dat` (or extra `xmu.dat`) introduces deterministic parity drift.
- Fixture-scoped regression compares every artifact present in the baseline directory; if baseline includes staged entry files such as `feff.inp`, stage the same file under `<actual-root>/<fixture>/<actual-subdir>/` to avoid false missing-artifact mismatches.
- For FULLSPECTRUM oracle fixtures, stage runtime inputs from compatibility baseline `FX-FULLSPECTRUM-001`; using oracle rewritten `xmu.dat` as the next input collapses input/output separation and introduces deterministic parity drift.
- Oracle parity requires a committed baseline file for every module-owned output artifact (including diagnostics like `logfullspectrum.dat`); otherwise comparator reports deterministic `Missing baseline artifact` mismatches.
- RIXS output contract now includes deterministic `rixs.sh`; keep the canonical script template at `crates/feff-core/src/modules/rixs/rixs.sh.template` and wire it through module output assertions (core parity + CLI oracle checks) when touching RIXS artifacts.
- Oracle baseline logs can be fixture-id sensitive; regenerate committed `*-ORACLE-*` baselines with the same fixture ID used by the oracle manifest to avoid deterministic `log*.dat` exact-text drift.
- Comparator policy is first-match: place artifact-scoped categories before broad wildcard categories to prevent tolerance bleed (for example, explicit SELF `sigma.dat` mapping ahead of thermal `**/sig*.dat`).
- Workflow-equivalent full-oracle parity currently runs canonical fixture IDs from `tasks/golden-fixture-manifest.json` (for example `FX-SELF-001`), so release summaries should report gate-level metrics from `artifacts/regression/oracle-report.json` instead of assuming `*-ORACLE-*` fixture IDs appear in that run.
- Lock Rust quality workflow contract changes with `crates/feff-cli/tests/quality_workflow_guard.rs`; if `.github/workflows/rust-quality-gates.yml` is intentionally edited, update the guard test in the same change.
- Lock Rust parity workflow contract changes with `crates/feff-cli/tests/parity_workflow_guard.rs`; if `.github/workflows/rust-parity-gates.yml` is intentionally edited, update the guard test in the same change.
- Migration-state docs should anchor to `tasks/oracle-16-module-migration-summary.md` and current parity artifacts (`oracle-report.json`, `oracle-diff.txt`, `oracle-summary.txt`, `oracle-stderr.txt`) instead of stale placeholder-era references.
- Cutover rehearsal evidence should be captured in timestamped directories under `artifacts/cutover-rehearsal/<YYYY-MM-DD>/<YYYYMMDD-HHMMSS>/` with one log per gate command plus smoke-output verification/hashes; the dated `tasks/cutover-rehearsal-YYYY-MM-DD.md` report should reference those exact paths and CI-equivalent command blocks.
